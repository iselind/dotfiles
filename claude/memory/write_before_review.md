---
name: Write files before showing for review
description: User prefers that files are written immediately, not shown as drafts first — they review via the IDE diff
type: feedback
---

Write files immediately rather than showing a draft in chat and waiting for approval before writing.

**Why:** The user can see the diff in the IDE — showing the draft in chat is redundant and adds unnecessary back-and-forth.

**How to apply:** In any skill or workflow that involves drafting documents (ADRs, plans, etc.), write the file first, then briefly note what was written and wait for feedback. If the user wants changes, edit in place.
